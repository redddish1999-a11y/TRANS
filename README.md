# P2P Trans Hub

基于浏览器、实时感知、多端同步的去中心化文件分发空间。使用 WebRTC（PeerJS）实现点对点传输，零服务器存储、高隐私、大文件不限速。

## 功能概览

- **配对**：发送端生成 3 个随机单词作为配对码（如 `tiger-blue-sky`），接收端输入即可连接。
- **发送池**：左侧拖拽或选择文件，显示发送总量；支持多文件。
- **接收簇**：右侧为每个已连接的接收者显示卡片，实时显示各文件下载进度。
- **接收端**：连接后可见待发文件列表，勾选后点击「下载所选」即可拉取；支持刷新列表。

## 技术要点

- **信令**：PeerJS（公共服务器 `0.peerjs.com`），Room ID = 配对码 = Peer ID。
- **传输**：DataChannel 可靠模式；文件按 16KB 分片，背压控制（缓冲区 >1MB 时等待）。
- **状态**：Zustand 维护 PeerStore（发送端）、远程文件列表与下载进度（接收端）。

## 本地运行

```bash
npm install
npm run dev
```

浏览器打开控制台提示的本地地址（如 `http://localhost:5173`）。建议开两个标签页分别作为「创建房间（发送端）」和「加入房间（接收端）」测试。

## 非本期范围

- 断点续传（刷新需重新开始）
- 历史记录（关闭即清空，保护隐私）

## 风险与对策

- **NAT 穿透**：依赖 STUN；对称 NAT 可能失败，可后续配置 TURN。
- **内存**：使用 `File.slice()` 流式分片，避免整文件进内存。
